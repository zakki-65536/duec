# GitHub初期設定、動作確認
## クローン
- コマンドプロンプトで、フォルダ`duec`を作りたいディレクトリに移動
  - ex) ディレクトリ構造を`hoge/dev/duec/sourcecode`としたい場合、`hoge/dev`に移動
- 以下のコマンドを実行
```
git clone https://github.com/zakki-65536/duec.git
```
- ディレクトリ`hoge/dev/duec`が作成される
## issue、pushの動作確認
### issueを作る
- GitHubのduecのトップページで、上部のツールバーからIssuesを開く
- 右上の「New Issue」をクリック
- タイトルを「{自分のユーザー名} 動作確認」とする
  - ex) 「zakki-65536 動作確認」
> [!IMPORTANT]
> 今後の実装では、issueのタイトルは「[FE] 追加する機能の内容」としてください。
> - フロントエンド: [FE]
> - バックエンド: [BE]
> - データベース作成: [DB]
- 右下の「Create」をクリック
- 右側のサイドバーの「Development」をクリックし、「create a branch」をクリック
- Branch nameは変更せず、「Create branch」をクリック
- 出てきたコマンドをコピー
```
git fetch origin
git checkout 1-zakki-65536-動作確認
```
### pushの動作確認
- Visual Studio Codeでフォルダ`duec`を開く
- 先ほどコピーしたコマンドを貼り付けて実行
- フォルダ`practice`内の自分のユーザー名のフォルダ内に`hello.py`を作成
- `hello.py`に以下を記載して保存
```python:hello.py
print('Hello GitHub')
```
- VS Code内でターミナルを開いて以下を実行してadd
```
git add .
```
- 次はcommit ({自分のユーザー名}の部分を埋めてください)
```
git commit -m "[test] first commit {自分のユーザー名}"
```
> [!IMPORTANT]
> 今後の実装では、[test]の部分は以下のいずれかにしてください。
> - feat: 新しい機能
> - WIP: 途中経過を一旦リモートブランチにマージしたいとき(issueは閉じない)
> - fix: バグの修正
> - docs: ドキュメントのみの変更
- 最後はpush (1行目は初回のみ必要)
```
git remote add origin https://github.com/zakki-65536/duec.git
```
```
git push -u origin
```
### プルリクエストを作成してmainブランチにマージ
- GitHubを開いてプルリクエストを作成
> [!IMPORTANT]
> 今後の実装では、プルリクエストが作成し終わった時点でグループメンバーに共有してください。
> 次はプルリクエストをmainブランチにマージする作業ですが、この作業の前に確認が必要です。
- プルリクエストをmainブランチにマージ
- mainブランチにマージできたら、今日の動作確認は終了です。お疲れ様でした。
