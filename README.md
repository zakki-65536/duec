# 開発用メモ
Gitのコマンドについては、[251119task.md](251119task.md)を参照しながら作業を進めてください。
## issueタイトルの規則
> [!IMPORTANT]
> 実装の際は、issueを作ってブランチを切ってから作業してください。  
> 進捗を把握しやすくするため、issueはできるだけガントチャートのタスクと対応させるようにしてください。

issueのタイトルは「[FE] 追加する機能の内容」としてください。
- フロントエンド: [FE]
- バックエンド: [BE]
- データベース作成: [DB]
## コミットメッセージの規則
```
git commit -m "[???] ○○のため、△△を追加"
```
[???]の部分は以下のいずれかにしてください。
- feat: 新しい機能
- wip: 途中経過を一旦リモートブランチにマージしたいとき(issueは閉じない)
- fix: バグの修正
- docs: ドキュメントのみの変更
